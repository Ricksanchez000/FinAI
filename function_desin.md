# FinAI · 今日分析工具 功能设计

> 目标：每个交易日收盘后 30 分钟内，产出一份**有观点、可回测、可点击下钻**的市场日报，覆盖 A 股（可扩展港美股）。差异化不在 LLM 文笔，而在 **数据广度 × 异动归因 × 历史相似度**。

---

## 1. 产品定位

| 维度 | 选择 | 理由 |
| --- | --- | --- |
| 用户 | 个人主动型投资者 / 量化研究员 | 既要"看得懂"，也要"挖得深" |
| 形态 | Web Dashboard + 每日邮件/微信推送 | 仪表盘交互 + 推送闭环 |
| 时效 | T 日 15:30 收盘 → 16:00 完成日报；盘中可选实时模式（v2） | 卡住"晚饭前一杯咖啡看完今天行情"的窗口 |
| 核心承诺 | **每个观点都给出数据出处和历史可比样本** | 反"玄学财经" |

---

## 2. 核心功能模块

### 2.1 市场总览 (Market Snapshot)
- 三大指数 / 申万一级行业涨跌幅热力图
- 涨跌家数、涨停/跌停、连板梯队、炸板率
- 北向资金净流入（行业 + 个股 Top 10）
- 两融余额、主力资金、ETF 申赎
- **亮点**：与近 60 个交易日的分布对比，标注当日处于多少分位（例：涨停数处于过去 60 日 92% 分位）

### 2.2 异动归因 (Anomaly Attribution)
对当日涨幅/跌幅/换手/资金流异常的标的，自动归因：
- **消息面**：抓取公告、研报、新闻、雪球热帖，LLM 抽取与个股关联度
- **板块联动**：是否跟随板块、板块龙头是谁
- **资金特征**：游资席位 vs 机构席位（龙虎榜）
- **历史相似度**：近 3 年内出现过哪些"形态/资金结构"相近的 K 线段，后续 5/10/20 日表现分布

### 2.3 板块轮动 (Sector Rotation)
- 申万一级 + 概念板块的当日强度排名
- 资金流入持续性（1日 / 3日 / 5日）
- 板块轮动图（Sankey 或散点：昨日强度 vs 今日强度）
- LLM 生成"轮动叙事"：从哪轮到哪、可能的宏观/事件驱动

### 2.4 个股深挖 (Stock Drilldown)
点击任一股票进入详情页：
- 基本面卡片（PE/PB/ROE/营收增速/机构持仓）
- 当日资金分时（主力净流入、大单、散户）
- 关联新闻/公告时间轴（与价格叠加）
- AI 摘要：今天为什么涨/跌、风险点、明日观察项

### 2.5 明日关注 (Tomorrow Watchlist)
- 重要事件日历（解禁、财报、宏观数据、美联储）
- 技术形态触发（突破、放量、金叉死叉）
- 自选股监控池
- LLM 综合输出 5 条"明日值得盯一眼的事"

### 2.6 历史回看 (Backtest Layer) ★ 差异化关键
- 所有 LLM 生成的观点都带 **可点击的"历史相似日"** 链接
- 给出在相似情境下，T+1 / T+5 / T+20 的市场表现分布（中位数 + 分位区间）
- 长期沉淀为"市场记忆库"，越用越值钱

## 3. 数据源

| 类型 | 来源 | 备注 |
| --- | --- | --- |
| 行情 | AkShare / Tushare / Wind（如有授权） | 优先 AkShare 起步，免费够用 |
| 龙虎榜、资金流 | AkShare、东方财富接口 | 免费但需做反爬退避 |
| 新闻 | 财联社、新浪财经、雪球（RSS / 抓取） | 注意 robots 与频次 |
| 公告 | 巨潮资讯网 (cninfo) | 官方源，PDF 需 OCR 或文本提取 |
| 研报 | 慧博 / 东方财富研报中心 | 摘要级别即可 |
| 宏观 | 国家统计局、央行、Wind 经济数据 | 低频，缓存即可 |
| 另类（v2） | 百度指数、微博热搜、谷歌趋势 | 情绪/关注度信号 |

## 4. 系统架构

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  Data Layer  │ → │  Signal Layer│ → │   LLM Layer  │
│  采集 / 清洗 │   │  指标 / 异动 │   │  归因 / 叙事 │
│  Postgres+S3 │   │  规则 / 因子 │   │  Claude/GPT  │
└──────────────┘   └──────────────┘   └──────────────┘
                                              ↓
                                     ┌──────────────┐
                                     │ Serving Layer│
                                     │  API + Web   │
                                     │  + 推送      │
                                     └──────────────┘
```

- **调度**：Prefect / Airflow（先用简单的 APScheduler 起步）
- **存储**：PostgreSQL（结构化）+ DuckDB/Parquet（行情快照）+ Redis（缓存）
- **向量库**：Qdrant 或 pgvector（新闻/公告/历史日报检索）
- **LLM**：Claude Sonnet 4.6（默认）+ Haiku 4.5（批量摘要降本）
- **前端**：Next.js + shadcn/ui + ECharts/TradingView Lightweight Charts
- **后端**：FastAPI（Python，与数据科学栈一致）
- **推送**：邮件（SMTP）+ 企业微信/飞书 Webhook + 可选 PWA 推送

---

## 5. LLM 用法约束（避坑）

1. **不要让 LLM 做数值计算**：所有数字来自 SQL / Pandas，LLM 只负责组织语言
2. **结构化输出优先**：先让 LLM 产出 JSON（事件、归因、风险），再用模板渲染中文段落，可控可回滚
3. **强制引用**：每段叙事必须挂数据源 ID，前端可"点进去看原始数据"
4. **缓存**：相同输入 → 相同输出，命中缓存直接返回；启用 Anthropic Prompt Caching 降本
5. **降级链路**：LLM 故障时退回模板化日报（指标 + 排行榜），不能开天窗

## 7. MVP 路线（建议 4 周）

**Week 1 · 数据底座**
- AkShare 接入：行情 / 涨跌停 / 龙虎榜 / 北向 / 资金流
- Postgres schema 设计 + 每日 ETL 任务
- 验收：能查到任意一天的全市场快照

**Week 2 · 信号层 + 单页 Dashboard**
- 指标计算：行业热力、连板梯队、资金分位
- Next.js + ECharts 出第一版总览页
- 验收：手动刷新能看到当日完整快照

**Week 3 · LLM 归因 + 异动榜**
- 异动榜单 + Claude 归因（接 Prompt Caching）
- 新闻/公告抓取 + 关联匹配
- 验收：随机抽 5 个异动股，归因不胡说

**Week 4 · 日报推送 + 历史相似度雏形**
- 邮件模板 + 调度
- 历史相似日 v0：基于行业涨跌幅向量的余弦相似
- 验收：连续 5 个交易日按时收到日报，准确率自检通过

**v2 候选**：盘中实时、个股深挖页、回测验证、移动端、量化 API

## 8. 风险与开放问题

- **数据合规**：抓取频次与 robots，必要时切付费源
- **LLM 幻觉**：通过结构化输出 + 数据引用强约束，但需要人工抽检评估集（建议 50 题黄金集）
- **评估指标缺失**：需要定义"日报质量"的可量化标准（如异动归因准确率、明日预判命中率）
- **成本**：每日 ~1000 次 LLM 调用，启用缓存后预计 < $1/日；规模化后再优化
- **个性化**：自选股 / 风格偏好如何沉淀为用户画像，影响推送内容
